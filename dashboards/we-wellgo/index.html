再現可能な開発プロセス
OpenSpec + skills 整備により、単なる速度改善ではなく「同じ速度で動き続けられる仕組み」を構築中。
指標の定義

計測指標

PR サイクルタイム
= merged_at − created_at（PR が open されてから merge されるまで）。ソースコードが書かれ PR として提出された後の時間のみを含み、実装前の設計・調査時間は含まない。本来の「リードタイム」（アイデア→本番デプロイ）とは異なる。
設計→実装完了
= archive コミット時刻 − artifacts 作成コミット時刻。OpenSpec の docs(openspec): add ... artifacts から chore(openspec): archive ... までの git コミットタイムスタンプ差分。設計ドキュメント作成から実装完了（archive）までを含む。
週次 PR 数
ISO week 単位でその週に merge された PR 数。祝日・休暇で谷ができる週がある。

データソース & 範囲

データソース
GitHub REST API（PR 一覧）＋ git log（openspec/changes ディレクトリのコミット履歴）。
対象 PR
メイン開発者（solo dev）が作成し merged 状態の PR のみ（計230+ 件）。Copilot bot PR や外部レビュアー PR は対象外。
設計→実装の母集団
